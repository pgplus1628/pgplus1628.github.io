利用eventfd实现多线程之间的event polling

Date Wed 17 February 2016 Tags [eventfd / parallel]

一个多线程之间同步的模型

很多时候，我们需要实现这样一个单机多线程之间的同步模型：有W个Worker，每个worker之间通过收发消息进行数据同步，我们称worker中收发消息的部分叫做Coordinator。每个Coordinator内部包括Receiver和Sender。Receiver负责接受W个Coordinator发送的消息，每接受到一个消息之后，出发一个event，执行用户定义的event_handler，worker会根据event做相应的计算。Sender负责向每个Coordinator发送消息。Receiver需要poll各个Sender发送来的消息。

这个模型在场景下都回用到，比如说要实现user space 的network protocol；比如把NUMA看作是分布式环境，每颗CPU看作是一个server，CPU之间通过收发消息通信(GraM: Scaling Graph Computation to the Trillions的做法)。

eventfd vs condition variable

pthread 中提供了condition variable，局限在于只能做1对1的同步，一个线程等待另外一个线程的signal，无法做到上述模型中的一个线程poll其他多个线程发送来的event。

eventfd是linux kernel在 2.6.22 之后提供的一个用于实现线程之间等待/通知的机制。

#include <sys/eventfd.h>
int eventfd(unsigned int initval, int flags);

eventfd创建了一个fd(file descriptor),这个fd内部维护了一个uint64_t类型的计数器，这个process可以read/write这个fd，在write(int fd, uint64_t val, ssize_t bytes)时，会使内部计数器加给定的val；在read时，如果counter非0,如果eventfd初始化的flags设置了EFD_SEMAPHORE，则会返回1，同时计数减1；若未设置EFD_SEMAPHORE,则直接返回计数器值，同时计数器清零。eventfd支持epoll等polling操作。 eventfd的更多信息

Implementation

上述多线程同步模型可以通过eventfd + epoll来实现。每个Receiver创建W个eventfd，每个Sender读取对应的eventfd. Sender通过写入eventfd来signal Receiver。 Receiver注册eventfd，通过epoll来监听事件。
